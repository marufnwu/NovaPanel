# Plan: Fix Missing API Error Messages in Web UI

## Problem

The web UI does not display toast notifications for API call errors. Users get no feedback when mutations (POST/PUT/DELETE/etc.) fail.

## Root Cause

Two separate issues combine to swallow all mutation error messages:

1. **`<ToastProvider />` is never mounted in the app tree** — The `ToastProvider` component (which renders Sonner's `<Toaster />`) exists in `components/ui/ToastProvider.tsx` but is **never imported or rendered anywhere** in the application. While the module-level toast listener IS registered, there is no `<Toaster />` mounted in the DOM to actually render the notifications on screen. This causes every `toast.error()` call (whether from the global handler or individual page `onError` callbacks) to silently do nothing visible.

2. **No global mutation error handler** — The `QueryClient` in `main.tsx` lacks a `defaultOptions.mutations.onError` handler. Most mutation hooks don't include their own `onError`, and many page components don't pass `onError` when calling `mutate()`. This means unhandled mutation errors are only stored in React Query's internal state and never surfaced to the user.

## Solution

Fix both issues:

1. **Mount `<ToastProvider />`** in `apps/web/src/main.tsx` (or `AppLayout.tsx`) so Sonner's `<Toaster />` actually renders in the DOM and can display toast notifications.

2. **Keep the global mutation `onError` handler** in `QueryClient` default options (already implemented in a previous turn). This catches all unhandled mutation errors and fires `toast.error()`.

## Implementation Steps

1. **Update `apps/web/src/main.tsx`**
   - Import `ToastProvider` from `./components/ui/ToastProvider`
   - Render `<ToastProvider />` inside the component tree (inside `QueryClientProvider` or `ThemeProvider`)

2. **Verify build passes**
   - Run `pnpm --filter web build` to ensure TypeScript compiles

3. **Smoke test**
   - Trigger a mutation error (e.g., try to create a site with invalid data) and confirm a toast appears

## Files to Change

- `apps/web/src/main.tsx` — import and mount `<ToastProvider />`

## Notes

- `ToastProvider.tsx`'s own comment says "Mount once at the root (AppLayout)" but this was never done. Mounting in `main.tsx` is equivalent and ensures coverage on all routes (including login).
- The Sonner `<Toaster />` renders fixed-position elements via a portal-like mechanism, so placing it at the root won't interfere with page layouts.
