### 5.6 Blank Screen Issue — FIXED ✅

**Status (2026-05-22)**: FIXED — Login page now renders correctly at `http://192.168.0.211:8732/`

**Final Root Cause**:
- Vite's `manualChunks` split React and vendor libraries into multiple chunks
- `@tanstack/react-query` (in `vendor-react-query`) contains its own copy of React 18's internals
- When `vendor-react-query` calls `n.Children` on its imported React (via `_e({__proto__:null,default:Re},[h])`), it was setting on its OWN React copy
- The main bundle (`index`) had ANOTHER copy of React via `import {r as wE ...} from"./vendor-react-query-mLZEUANs.js"`
- When the bundled React tried to use `Children` (set via the `n.Children` assignment), the bundled React's internals were undefined because `n` was the vendor-react-query's React, not the main bundle's React

**The Fix**:
Simplified `manualChunks` to put ALL `node_modules` into a single vendor chunk:
```js
manualChunks: (id) => {
  if (!id.includes('node_modules')) return undefined;
  return 'vendor';
}
```
This ensures only ONE copy of React exists across the entire bundle.

**Commits**:
- `3ca9fc9`: React 19→18 downgrade + COOP/CORP headers + install.sh fixes
- `832b6e9`: Added `dedupe: ['react', 'react-dom']` (insufficient alone)
- `4bc16e1`: Simplified chunking to single vendor chunk (definitive fix)
