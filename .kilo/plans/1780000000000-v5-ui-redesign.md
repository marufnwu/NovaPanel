# Plan: NovaPanel v5 — Complete UI/UX Redesign

## Goal
Rebuild the entire NovaPanel frontend with a modern, minimal SaaS dashboard aesthetic:
- **Top navigation** with global search / command palette
- **New color scheme** (zinc neutrals + emerald accent)
- **All 30+ pages** redesigned from scratch in one pass
- **Responsive** across desktop, tablet, and mobile

---

## Current State
- **Stack**: React 18 + Tailwind CSS 3 + Radix UI + TanStack Router/Query + shadcn/ui
- **Layout**: Sidebar (w-56) + TopBar + Breadcrumb + Main content
- **Pages**: 30+ pages across domains, sites, databases, mail, firewall, etc.
- **Routing**: Code-based TanStack Router with ~35 routes
- **Font**: Geist variable
- **Icons**: Lucide React

---

## Design Direction

### Aesthetic
- Minimal, clean SaaS dashboard (inspired by Vercel, Railway, Linear)
- High whitespace, clear hierarchy
- Subtle borders (1px zinc-200), soft shadows
- Card-based content containers with rounded corners

### Layout Shift
| Current | v5 |
|---------|-----|
| Sidebar (left, w-56) | Top nav bar (h-14) |
| Collapsible nav groups | Horizontal nav with dropdowns |
| Back links in sidebar | Breadcrumbs + "Back to" links in page headers |
| Org switcher in sidebar | Org switcher in top-right dropdown |
| TopBar (right actions) | Integrated into top nav |

### New Color Scheme
| Token | Value | Usage |
|-------|-------|-------|
| `background` | zinc-50 (light) / zinc-950 (dark) | Page background |
| `foreground` | zinc-900 (light) / zinc-100 (dark) | Primary text |
| `primary` | emerald-600 | CTAs, active states, accent |
| `primary-foreground` | white | Text on primary |
| `muted` | zinc-100 (light) / zinc-900 (dark) | Secondary backgrounds |
| `muted-foreground` | zinc-500 | Secondary text |
| `card` | white (light) / zinc-900 (dark) | Content cards |
| `border` | zinc-200 (light) / zinc-800 (dark) | Borders |
| `ring` | emerald-500 | Focus rings |
| `accent` | emerald-50 (light) / emerald-950 (dark) | Hover highlights |

### Typography
- Keep Geist for body/UI text
- Add a display font for page headings (optional: Geist Mono for data/code)
- Scale: xs (12px), sm (13px), base (14px), lg (16px), xl (20px), 2xl (24px), 3xl (30px)

## Execution Phases

### Phase 1: Design System Foundation
1. **Update `tailwind.config.js`**
   - Replace color variables with new zinc + emerald scheme
   - Update border-radius tokens (keep lg=8px, add xl=12px)
   - Add new spacing tokens if needed
   - Update font settings

2. **Update `index.css` CSS variables**
   - Redefine all `--background`, `--foreground`, `--primary`, etc.
   - Ensure smooth dark mode transition

3. **Install new dependencies**
   - `framer-motion` (for subtle page transitions, drawer animations)
   - `@fontsource-variable/geist-mono` (if adding mono font)

### Phase 2: Layout System (The Shell)
1. **Delete**: `Sidebar.tsx`, `TopBar.tsx`, `AppLayout.tsx` (old versions)
2. **Create new `AppLayout.tsx`**
   - Top nav bar (h-14, fixed or sticky)
   - Logo left, nav links center, user/org actions right
   - Command palette trigger (Cmd+K button)
   - Mobile: hamburger menu → drawer
   - Main content area below nav (no sidebar)

3. **Create new `TopNav.tsx`**
   - Horizontal navigation with grouped dropdowns
   - Active state indicator (bottom border or bg highlight)
   - Collapsible on mobile → hamburger drawer

4. **Create `CommandPalette.tsx`**
   - Cmd+K / Ctrl+K global search
   - Search across pages, actions, and entities
   - Recent items, fuzzy matching
   - Built on top of existing `cmdk` dependency

5. **Create `MobileNavDrawer.tsx`**
   - Slide-in drawer for mobile navigation
   - All nav groups as collapsible sections

6. **Create `PageShell.tsx`**
   - Reusable page wrapper: padding, max-width, responsive
   - Optional: page header slot (title + actions)
   - Optional: tabs slot for tabbed pages

7. **Update `Breadcrumb.tsx`**
   - Simpler, more minimal style
   - Optional: move inside page header instead of separate bar

### Phase 3: Component Library Refresh
Restyle all existing shadcn/ui components with new tokens:
- `Button` — new primary color, sharper focus rings
- `Card` — softer shadow, new border color
- `Dialog` / `Sheet` — updated backdrop, radius
- `Input` / `Textarea` / `Select` — new border/focus styles
- `Badge` — new color variants (emerald, amber, rose, slate)
- `Table` — cleaner borders, hover states
- `DropdownMenu` — updated to match new theme
- `Tabs` — new active indicator style
- `Toast` / `Sonner` — updated colors
- `Skeleton` — updated shimmer colors

Add new utility components:
- `PageHeader` — title + subtitle + action buttons
- `StatCard` — metric display with trend indicator
- `DataTable` — wrapper around TanStack Table with new styling
- `EmptyState` — updated illustration + CTA
- `SearchInput` — styled search with icon
- `StatusBadge` — colored status indicators (online, offline, warning, error)
- `StatusDot` — small colored dot with tooltip

### Phase 4: Core Pages (Start Here)
Redesign these 6 pages first (most heavily used):
1. **LoginPage** — centered card, cleaner form, better error states
2. **DashboardPage** — stats grid (4 cards), recent activity feed, quick actions, server health overview
3. **SitesPage** — table view with status badges, search, filters, action dropdowns
4. **SiteDetailPage** — tabbed layout (overview, settings, logs, backups)
5. **DomainsPage** — card grid or table with status indicators
6. **DatabasesPage** — similar to Sites pattern

### Phase 5: Secondary Pages (Continue)
Redesign all remaining pages following the new patterns:
- **Web**: SSL, DNS, Web Server, PHP, Cloudflare
- **Services**: Mail, FTP, Cron, Firewall
- **System**: Files, Terminal, Security, Storage, Containers, Registries
- **Tools**: Backups, Installer, Notifications, Audit, Billing, Webhooks, Plugins, Jobs
- **Settings**: Profile, API Tokens, Server Settings, Organizations
- **Auth**: Reset Password
- **Detail pages**: DatabaseDetailPage

### Phase 6: Polish & Final Integration
1. **Animations**
   - Page transitions (subtle fade/slide)
   - Modal/dialog open/close
   - Mobile drawer slide
   - Skeleton loading states
   - Stats card number count-up

2. **Responsive pass**
   - All pages work on tablet (md) and mobile (sm)
   - Tables become card lists or horizontal scroll on mobile
   - Forms stack vertically on mobile
   - Detail pages use tabs instead of side-by-side layouts

3. **Dark mode**
   - Verify all new colors invert correctly
   - Check contrast ratios
   - Test on OLED (pure black vs zinc-950)

4. **Performance**
   - Verify bundle size didn't explode
   - Check for any layout shift
   - Ensure no FOUC on theme switch

5. **Build & Test**
   - `pnpm build` passes 3/3
   - TypeScript strict mode clean
   - Smoke test all routes render without crash

## Constraints
- Do NOT change API routes or backend logic
- Do NOT modify `router.tsx` route definitions (only component imports if needed)
- Keep existing data fetching hooks (`useSites`, `useDomains`, etc.) — only change presentation
- Keep existing forms and validation logic — only change styling
- Maintain all existing functionality (CRUD operations, modals, confirmations)
- Follow React 18 + TypeScript strict mode
- No breaking changes to the API client (`apps/web/src/api/client.ts`)
