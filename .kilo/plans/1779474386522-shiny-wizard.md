# NovaPanel UI Full Rewrite — Design-Driven Implementation Plan

## Source of Truth
This plan follows the design document exactly. Every visual decision, spacing value,
color usage, and interaction pattern must match the spec. When in doubt, re-read
the design document — not this plan.

---

## Approach: Full Rewrite From Scratch

All existing UI components and pages will be replaced. The backend API, auth system,
Zustand stores, TanStack Query hooks, and WebSocket logic remain untouched.

**What stays:**
- API layer (api client, hooks, services)
- Auth system, Zustand stores, QueryClient config
- WebSocket logic (/ws/jobs)
- Router definitions (routes may change but router stays)

**What gets rewritten:**
- All layout components (AppLayout, Sidebar, Topbar, StatusBar)
- All shared components (Card, Button, Table, Modal, Toast, etc.)
- All page components (~30 pages)
- Global styles, CSS variables, theme

---

## Phase 1 — Design System Foundation

### 1.1 CSS Variable System
Create the complete color, spacing, typography, and border token system.
All components must use these tokens. No hardcoded values anywhere.

Variables to define:
- Color tokens: background-primary/secondary/tertiary, text-primary/secondary/tertiary,
  border-primary/secondary/tertiary, success/warning/danger/info light/dark/mid
- Spacing: 4px, 8px, 12px, 16px, 24px, 40px
- Typography: 22px/500, 18px/500, 16px/500, 14px/400, 13px/400, 12px/500, 11px/500
- Border radius: 4px, 6px, 8px, 12px, 99px
- Border: 0.5px solid var(--color-border-tertiary) default
- Font: system sans-serif + monospace for code
- Font weights: 400 and 500 only

### 1.2 Global Styles
- Reset/normalize
- Base body styles
- Focus ring: box-shadow 0 0 0 3px rgba(color, 0.15)
- No gradients, no shadows, no glow effects
- Dark mode via CSS variables (not separate stylesheets)

### 1.3 Tabler Icons
Replace all Lucide icons with Tabler Icons outline style (18px in nav, 15px in buttons, 16px in toasts).

## Phase 2 — Layout Shell

### 2.1 AppLayout
Create the master layout wrapper:
```
Sidebar (fixed left, 220px expanded / 52px collapsed)
├─ Topbar (48px fixed)
├─ StatusBar (36px fixed)
└─ Content (scrollable, 24px padding)
```

### 2.2 Sidebar
- Grouped navigation per design doc (Apps, Server, Domains, System, Settings)
- Group labels: 11px uppercase, letter-spacing 0.08em
- Nav item states: default, hover, active (left border 2px solid)
- Collapsible groups, state persisted to localStorage
- Collapsed mode: icons only, tooltips
- Active state: parent route active for nested routes

### 2.3 Topbar
- Left: breadcrumb (Section / Page / Resource)
- Right: action buttons + notification bell + user avatar
- 48px height, border-bottom 0.5px

### 2.4 StatusBar
- Always visible, 36px height
- CPU/RAM/Disk percentages with color thresholds
- Service dots (Nginx, MySQL, PHP) — green/red/amber
- Poll GET /stats/server every 15s
- Clickable: stats → /monitoring, services → /services

### 2.5 Breadcrumb
- Mounted in AppLayout or PageHeader
- Format: Section / Page / Resource name
- Links for all segments except current page

## Phase 3 — Shared Components (Atomic)

Build each component from scratch per design doc. All must use CSS variables.

### 3.1 Button
- Sizes: default (32px), small (26px), large (38px)
- Variants: primary, default, danger, ghost, loading
- Icon buttons: 32px/26px square, 6px border-radius
- Hover: bg shift. Active: scale(0.98). Disabled: opacity 0.5

### 3.2 Form Inputs
- Height 34px, padding 6px 10px, 13px font
- Border 0.5px solid, border-radius 6px
- Label above input, 12px weight 500
- Error replaces hint, never both
- Focus ring: box-shadow 0 0 0 3px rgba(color, 0.15)

### 3.3 StatusBadge
- Structure: [dot 6px] [label 12px weight 500]
- Padding 3px 10px, border-radius 99px
- Color mapping: running (success), stopped (danger), pending (warning), deploying (info + pulse)
- Always include dot. Never text-only.

### 3.4 Card
- Background primary, border 0.5px, border-radius 12px, padding 16px
- Optional header: title 16px/500 left, action right, 12px mb
- Optional sections separated by border-top

### 3.5 StatCard
- Background secondary, border-radius 8px, padding 14px 16px
- Label 12px muted, Value 24px/500, Sub 11px muted
- Threshold colors for value

### 3.6 DataTable
- Wrapper: border 0.5px, border-radius 12px, overflow hidden
- Header: 11px/500 uppercase, letter-spacing 0.04em, muted color
- Rows: 14px, padding 10px 16px, hover bg secondary
- First column: primary text weight 500
- Actions column: right-aligned icon buttons only
- Empty state: EmptyState component inside wrapper
- Loading: 5 skeleton rows, pulse animation
- Pagination below table: "Showing X–Y of Z" + Prev/Next

### 3.7 Modal
- Overlay: rgba(0,0,0,0.4)
- Panel: bg primary, border 0.5px, border-radius 12px, padding 24px
- Sizes: 320px (confirm), 480px (default), 640px (large)
- Header: title 16px/500 + close × button
- Footer: right-aligned, Cancel leftmost, Primary rightmost
- Close on overlay click, Escape key, × button
- Focus trapped, body scroll locked
- Never nest modals

### 3.8 ConfirmDialog (destructive)
- 360px width, border 0.5px danger color
- Friction levels: low (just confirm), medium (confirm + text), high (type "DELETE")
- High-impact: input field, button disabled until "DELETE" typed exactly
- Cancel auto-focused on open

### 3.9 Toast
- Top-right, 16px from edge, max 3 visible
- Border-left 2px semantic color, border-radius 8px
- Auto-dismiss: success 4s, info 5s, warning 6s, error NEVER
- Specific messages only: "SSL issued for myapp.com" not "Success"

### 3.10 EmptyState
- Centered, padding 48px 24px
- Icon 32px muted, Title 15px/500, Description 13px secondary, Action button optional
- Dashed border 0.5px, border-radius 12px
- Contextual action if possible

### 3.11 Skeleton
- Pulse animation 1.4s ease-in-out infinite
- Match real layout exactly
- Replace entire component, not parts
- Max 10s then show error with retry

### 3.12 ActivityFeed (slide-over drawer)
- 360px width, full height, slides from right
- Items: icon 18px, title 13px/500, subtitle 12px muted, time 11px muted
- Close on overlay click, Escape, or "View all" → /jobs
- Bell icon in topbar with badge (blue=running, red=failed)

## Phase 5 — Final Polish & Cleanup

### 5.1 Remove Old Components
Delete all old components that have been replaced:
- Old AppLayout, Sidebar, Topbar
- Old Card, Button, Input, Table, Modal, Toast, EmptyState, Skeleton
- Old DataTable (migrate any remaining usages to new DataTable)

### 5.2 Verify Checklist
For every component and page, verify:
- [ ] Uses only CSS variables for color
- [ ] Font weight only 400 or 500
- [ ] All interactive elements have focus states
- [ ] Loading state (skeleton or button spinner)
- [ ] Empty state (if shows list)
- [ ] Error state (if fetches data)
- [ ] Async: loading → success toast → cache invalidation
- [ ] Errors: persistent toast with specific message
- [ ] No fake/simulated data
- [ ] Dark mode works
- [ ] Icon-only buttons have aria-label
- [ ] Decorative icons have aria-hidden="true"
- [ ] Reused existing components
- [ ] Tab state in URL query param
- [ ] Confirm dialogs use correct friction level

### 5.3 End-to-End Test
Critical path: login → create site → deploy → check monitoring → create backup

## RULES (from design doc — non-negotiable)

- Never hardcode hex colors. Always CSS variables.
- Never use color for decoration.
- No gradients, shadows, glow, noise, mesh.
- Font weights: 400 and 500 only.
- Sentence case everywhere. Uppercase only for 11px section labels.
- No mid-sentence bolding.
- Monospace for: logs, code, terminal, IPs, commands, paths, config values.
- Line height: 1.6 body, 1.3 headings.
- Spacing: 4, 8, 12, 16, 24, 40px only.
- Border radius: 4, 6, 8, 12, 99px only.
- Border: 0.5px solid (except featured cards: 1.5px, focus rings: 3px box-shadow).
- No drop shadows. No box-shadow except focus rings.
- No animation beyond skeleton→content.
- No parallax, scroll-triggered animations, decorative motion.
- No simulated or fake data.
- No localStorage for server data.
- No spinner for page-level loading (use skeleton).
- No "Something went wrong" messages.
- No auto-dismissing error toasts.
- No modal stacking.
- No optimistic updates on destructive actions.
- No full page reload for navigation.
- No inline styles for colors.
- No emoji in UI.
- No more than 3 toasts simultaneously.
- No polling intervals shorter than 2 seconds.

## Deployment Workflow

**Preferred approach (git-based):**
1. Commit all changes locally with descriptive messages
2. Push to remote repository
3. On server: `git pull` then run rebuild script
4. Server has existing `scripts/rebuild-and-deploy.sh` for this purpose

**Do NOT upload files directly via SFTP** unless explicitly requested.
Use the server's git-based workflow for all deployments.
