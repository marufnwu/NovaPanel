# NovaPanel — Full UI Rewrite Plan

## Scope

**DELETE ALL EXISTING UI.** Every component, every page, every style, every layout.
Keep only: API client, auth system, Zustand stores, TanStack Query hooks, WebSocket logic, router config.

**Rebuild from scratch** following the design document exactly.

## Transition Strategy: Big Bang (Option 2)

1. **Delete everything** in one operation: all components, pages, styles, layout files
2. **Start from zero**: empty `apps/web/src/` with only `api/`, `store/`, `lib/utils.ts` kept
3. **Build up sequentially**: Phase 1 → Phase 2 → Phase 3 → Phase 4 batches
4. **Panel will be non-functional** until at least LoginPage + AppLayout + DashboardPage are built
5. **Commit after each phase** for rollback safety

## Phase 3 — Atomic Components (~20 components)

### Button
- Sizes: default (32px), small (26px), large (38px)
- Variants: primary, default, danger, ghost, loading
- Icon buttons: 32px/26px square
- Hover: bg shift. Active: scale(0.98). Disabled: opacity 0.5

### Input
- Height 34px, padding 6px 10px, 13px font
- Border 0.5px solid, border-radius 6px
- Label above: 12px/500
- Error replaces hint
- Focus ring: box-shadow 0 0 0 3px rgba(color, 0.15)

### Select, Textarea, Checkbox, Radio
- Same border/radius as Input
- Textarea: min-height 80px, resize vertical
- Checkbox: 16px, checked filled with primary

### Card
- Background primary, border 0.5px, border-radius 12px, padding 16px
- Optional header: title 16px/500 left, action right
- Optional sections separated by border-top

### StatCard
- Background secondary, border-radius 8px, padding 14px 16px
- Label 12px muted, Value 24px/500, Sub 11px muted
- Value color changes at thresholds

### StatusBadge
- [dot 6px] + [label 12px/500]
- Padding 3px 10px, border-radius 99px
- Colors: running (success), stopped (danger), pending (warning), deploying (info + pulse)
- Always include dot

### DataTable
- Wrapper: border 0.5px, border-radius 12px, overflow hidden
- Header: 11px/500 uppercase, letter-spacing 0.04em, muted
- Rows: 14px, padding 10px 16px, hover bg secondary
- First column: primary text weight 500
- Actions: right-aligned icon buttons
- Empty state inside wrapper
- Loading: 5 skeleton rows
- Pagination: "Showing X–Y of Z" + Prev/Next

### Tabs
- Tab bar: border-bottom 0.5px
- Tab: 13px, padding 10px 16px
- Active: primary text, weight 500, border-bottom 1.5px solid, margin-bottom -0.5px
- Content: 24px padding-top
- Max 9 tabs
- Tab state in URL query param

### Modal
- Overlay: rgba(0,0,0,0.4)
- Panel: bg primary, border 0.5px, border-radius 12px, padding 24px
- Sizes: 320px (confirm), 480px (default), 640px (large)
- Header: title 16px/500 + close ×
- Footer: right-aligned, Cancel left, Primary right
- Close on overlay click, Escape, ×
- Focus trapped, body scroll locked
- Never nest

### ConfirmDialog
- 360px width, border 0.5px danger color
- Low impact: confirm button only
- Medium impact: confirm + warning text
- High impact: type "DELETE" input + disabled button until matched
- Cancel auto-focused

### Toast
- Top-right, 16px from edge, max 3
- Border-left 2px semantic color, border-radius 8px
- Auto-dismiss: success 4s, info 5s, warning 6s, error NEVER
- Specific messages only

### EmptyState
- Centered, padding 48px 24px
- Icon 32px muted, Title 15px/500, Description 13px, Action button
- Dashed border 0.5px, border-radius 12px
- Contextual action if possible

### Skeleton
- Pulse animation 1.4s ease-in-out infinite
- Match real layout
- Replace entire component
- Max 10s then error with retry

### ActivityFeed (slide-over drawer)
- 360px width, full height, slides from right
- Items: icon 18px, title 13px/500, subtitle 12px, time 11px
- Close on overlay click, Escape
- Bell icon in topbar with badge (blue=running, red=failed)

### LoadingPage
- Full-page skeleton layout
- Used while page-level data loads

### PageHeader
- Title 22px/500 + optional description 14px muted + action button right
- 24px margin-bottom
- If tabs below: 16px margin-bottom

## Deployment Workflow
